Why it stays useful

Keep recurring guidance in reach.

The problem is simple: useful operating habits keep getting rediscovered, then lost again. CSBP gives them a controlled place to live so they can stay available across Codex sessions without becoming automatic sprawl.

Without CSBP

With CSBP

What changes

Useful guidance stops relying on luck.

You get a narrow shared layer for habits that matter, instead of hoping they resurface later.

How it stays controlled

Built in three files, held in order.

CSBP works because it separates activation, runtime practice, and formation logic. The system stays understandable because each file has one job and the authority order is explicit from the start.

Core files

CSBP-entry-point.txt

Turns the system on, sets authority order, and routes the correct mode.

shared-best-practice.txt

Stores promoted runtime guidance only. Active entries apply when relevant.

shared-best-practice-compiler.txt

Evaluates, rewrites, promotes, deprecates, rejects, and removes candidate practices during formation work.

Lifecycle

Nothing becomes active by accident.

A candidate practice has to earn promotion. It must recur, stay actionable, remain compatible with stronger instructions, and be approved by the operator before it becomes active runtime guidance.

01Notice

A repeated operational issue or useful practice appears.

02Evaluate

The compiler checks recurrence, scope, actionability, and authority.

03Normalize

Valid material is rewritten into a small runtime block.

04Approve

The operator accepts, edits, or rejects the candidate.

05Apply

Only approved active entries may guide future work when relevant.

What it prevents

Useful practice without memory sprawl.

CSBP is deliberately conservative. It avoids automatic capture, silent rule creation, and background promotion. It gives useful practice a path without giving chat history the power to rewrite future behavior.

Good fit

Recurring, actionable, preventive.

Best for guidance that applies across sessions, reduces repeated waste, and can be used without lengthy interpretation.

Bad fit

One-off, local, vague, or stronger elsewhere.

Rejected if it belongs in a specific project, duplicates stronger instruction, or cannot guide real work.

Short framing

What CSBP is, and what it is not.

CSBP is a portable protocol kit for shared, domain-agnostic operational best practice across Codex environments and sessions. It is not a plugin, package, hosted service, or auto-memory layer.

Its job is smaller and more useful: give approved recurring guidance a stable place to live without replacing the host contract.
