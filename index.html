Why it exists

Keep recurring guidance usable.

In long-lived Codex setups, useful operational lessons often land in the wrong layer: trapped in chat history, pushed too early into AGENTS.md, or mixed into automatic memory. CSBP creates a middle layer that is explicit, reviewable, and operator-controlled.

Without CSBP

Signal decays in conversation.

Recurring mistakes keep reappearing because the guidance that could prevent them never gets a stable runtime home.

With CSBP

Only promoted patterns become active.

The operator and Codex can discuss candidates freely, but nothing becomes live runtime guidance without an explicit approval path.

Design intent

Prevent drift by architecture.

The separation between bootstrap, runtime, and compiler logic is the mechanism that keeps the system narrow and trustworthy.

Mechanics

Three files, different authority, fixed load order.

CSBP is not a note bucket and not a replacement for the host contract. It works because each file has one job, the authority order is explicit, and the compiler does not leak into runtime use.

Bootstrap

CSBP-entry-point.txt

Read first. Activates CSBP, establishes authority order, mode routing, load order, and write restrictions.

Runtime

shared-best-practice.txt

Stores promoted practices only. Active entries apply when relevant; deprecated entries stay out of runtime behavior.

Formation

shared-best-practice-compiler.txt

Evaluates, normalizes, revises, promotes, deprecates, removes, or rejects candidate practices during formation work.

Lifecycle

Candidate to runtime, with a gate in the middle.

The protocol does not treat every useful observation as a permanent rule. It runs candidates through evaluation and normalization first, then requires operator approval before promotion.

01 Observe A recurring issue, omission, or pattern enters discussion.
02 Evaluate Recurrence, scope, usefulness, and overlap with stronger rules are checked.
03 Normalize The idea is rewritten into a short, preventive, runtime-ready instruction.
04 Approve The operator decides whether the candidate is allowed into runtime.
05 Promote The practice lands in the runtime file with its final identifier.

Short framing

What CSBP is not.

It is not a plugin, package, CLI, hosted service, auto-memory layer, or replacement for AGENTS.md.

It is a portable, reviewable, AI-only protocol kit for shared operational best practices across Codex sessions.
